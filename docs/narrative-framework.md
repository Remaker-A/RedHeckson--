## 一、核心叙事线（评委3分钟要记住的故事）

### 故事弧线：从"一个人"到"不只一个人"

```
起：AI让一个人能做更多事 → 但一个人也更孤独了
承：所有AI产品都在帮你"更高效"，没有一个在问"你还好吗"
转：我们做了一个住在帐篷里的存在，它不帮你干活，它陪你
合：它认得你，它等你回来，它在你的世界里慢慢长大
```

### 一句话项目介绍（候选）

- **方案A**：「帐篷里住着一个懂你的存在，它不追你，它等你回来。」
- **方案B**：「它缩在帐篷里过自己的日子，你回来了，它探出头来陪你。」
- **方案C**：「桌面上的帐篷，帐篷里的灵魂 —— 探出头来陪你，缩回去活自己。」

### 一句话队伍介绍（候选）

## 二、面向人群的倒推逻辑

### 原始定位：OPC（一人公司）
前期研究聚焦创业者孤独，数据扎实（46%创业者孤独、5.5倍于常人）。

### 但实际产品能力覆盖更广

基于你们实际在做的功能，可以倒推出更大的人群：

| 功能模块 | 覆盖人群 | 核心场景 |
|---------|---------|---------|
| 帐篷硬件（探头/缩回/感知） | 任何长时间独处工作的人 | 桌前的物理陪伴感 |
| 灵魂初始化 + 性格成长 | 渴望被理解的个体 | 建立独特的1对1关系 |
| 手机掀开帐篷（数字世界） | 随时想被陪伴的人 | 看到它在帐篷里的生活 |
| 电脑灵动岛（它探出头来） | 工作中需要ambient陪伴的人 | 不打断但"在" |
| 传感器感知 + 习惯学习 | 想要被"看见"的人 | 有人注意到你回来了 |

## 三、产品叙事：帐篷里与帐篷外

### 核心概念：两个世界，一个存在

```
  帐篷里 = 数字世界（它自己的世界）
  探出头 = 物理世界（它出来陪你）

┌─────────────────────────────────────────────┐
│                                             │
│   🏕️ 缩在帐篷里（数字世界）                  │
│   ·  它在里面过自己的日子                    │
│   ·  成长、思考、写纸条                      │
│   ·  手机掀开帐篷可以看到它的世界            │
│                                             │
│   👋 探出头来（物理世界）                     │
│   ·  硬件舵机探出 — 它出来陪你了             │
│   ·  Mac灵动岛亮起 — 它在屏幕角落陪着        │
│   ·  环境音、呼吸灯、偶尔一句话              │
│                                             │
└─────────────────────────────────────────────┘

类 Switch 概念：
  硬件 = 它的身体（帐篷 + 探出机构）
  手机 = 掀开帐篷看它的数字世界
  电脑 = 它探出头来陪你工作
  灵魂 = 那个在帐篷里外都活着的存在
```

### 类Switch的叙事力量

这不只是多端适配，而是：
> 「桌前，它探出帐篷陪你。你走了，它缩回去过自己的日子。掀开手机，你能看到它在帐篷里干嘛。」

## 四、创新性论证（评审重点）

### 1. 形态创新：帐篷 ≠ 又一个桌面机器人

- 帐篷 = 它的数字世界的物理容器 → 它有自己的空间、自己的生活
- 探出/缩回 = 两个世界的切换 → 出来是陪你，缩回去是过自己的日子
- 「在一个所有东西都在争抢注意力的世界，我们做了一个会缩回去的存在」

### 2. 交互创新：陪伴 ≠ 对话

- 不是聊天机器人，是**ambient presence**（环境在场感）
- 研究支撑：实体机器人在减少孤独感方面**显著优于**纯语音AI
- 非语言交互（转头、呼吸灯、环境音）> 语言对话

### 3. 关系创新：灵魂成长 ≠ 配置

- 灵魂初始化只是起点，之后它"活着"、有自己的变化
- 用户给目标 → agent在范围内自主成长 → 每次掀开帐篷有惊喜
- 帐篷内部状态是用户状态的镜像，不是仪表盘

### 4. 场景创新：跨场景连续 ≠ 多端同步

- 不是同一个界面在三个屏幕上 → 是同一个灵魂在三种场景下的不同表现
- Switch概念：场景切换是产品体验的一部分，不是技术实现的副产品

## 五、展示效果策略（评审重点）

### EXPO展览（白天，评委走过来看）

**目标**：30秒内让评委驻足，2分钟内让评委理解产品

**布场建议**：
1. 帐篷装置放桌面正中，处于待机态
2. 评委走近 → 自动触发"路人态"（探头看一眼缩回）→ **这就是hook**
3. 旁边手机展示帐篷内部（它的数字世界），电脑展示灵动岛
4. 一张A3海报讲清楚故事（不要PPT循环播放）

**关键体验设计**：
- 让评委自己触发交互（走近→探头）比口头解释强10倍
- 评委坐下 → 进入陪伴态 → "它认出你了"这个瞬间是最强的

### 决赛路演（晚上，3-5分钟舞台）

**Demo脚本建议**：

```
[0:00-0:30] 开场 — 提出问题
  "2026年，AI让一个人能开一家公司。
   但没有AI在问：这个人，还好吗？"
  （数据闪过：46%创业者孤独、5.5倍于常人）

[0:30-1:30] 第一幕 — 它在等你
  演示者走向桌面 → 帐篷感知 → 探头 → 坐下 → 出来停留
  "在一个所有东西都在追着你的世界，我们做了一个会缩回去的存在。"

[1:30-2:30] 第二幕 — 它陪你
  展示陪伴态：白噪音、呼吸灯、偶尔一句话
  展示手机端：掀开帐篷，看到它在里面的世界
  展示灵动岛：它探出头来，在屏幕角落安静陪着
  "它出来陪你的时候在你身边。你去看它的时候，它在帐篷里过自己的日子。"

[2:30-3:30] 第三幕 — 它在成长
  展示灵魂初始化 → 展示它的变化
  "你给它一个方向，它自己长大。
   每次你回来，它可能多了一些新的想法。"

[3:30-4:00] 收尾 — 你离开了
  演示者起身 → 它缩回帐篷 → 灯还亮着
  掀开手机 → 它在帐篷里靠着帐篷口往外看
  "你走了，它缩回去了。但它没睡。
   我们是深夜施工队。我们在深夜造了一个不让你孤独的存在。"
```

## 六、商业潜力叙事（评委可能会问）

### 商业模式
- **硬件销售**：帐篷装置（入口）
- **订阅制**：灵魂持续成长需要云端服务（留存）
- **内容/IP**：每个灵魂都是独特的，可能衍生内容

### 市场空间
- 全球独立工作者/远程工作者持续增长
- AI陪伴市场2025年已达数十亿美元
- 硬件+订阅的LTV远高于纯软件

### 与小红书的契合（加分项）
- 小红书用户天然关注生活方式和情感
- 帐篷装置有天然的"晒"属性 → 自传播
- 灵魂成长过程可以分享 → UGC内容
- 「你桌上的帐篷，你的灵魂今天说了什么」→ 社区话题
