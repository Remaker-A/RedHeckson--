## 一句话定义

**桌上有一个小房子，里面住着一个懂你的存在。**

---

## 我们看见的人

### 不是所有人，是这一群人

独立开发者、一人公司创业者、自由职业者、远程工作者、独立创作者。

他们的共同点不是职业，而是处境：

- 一个人面对所有决策、所有等待、所有深夜
- AI让他们一个人能干更多事，但也让他们更长时间独自承受
- 工具越来越强，但没有任何系统在乎"人还撑不撑得住"

### 他们真正的问题

不是效率低。是三件更深的事：

**决策孤独** — 所有选择都要自己扛，没有人帮你验证另一条路会怎样。研究显示46%的创业者面临孤独和隔离感，创业者经历孤独的可能性是普通人的5.5倍。

**状态断裂** — 不是不知道该做什么，而是在高压和低谷之间反复切换，每次都要重新进入状态。心理学称之为"情绪鞭打"。

**无声消耗** — 看起来在工作，其实在慢慢被掏空。没有人注意到你回来了，也没有人注意到你快要撑不住了。

### 我们在回应的社会现实

> AI时代放大了个体能力，但没有解决个体劳动的孤独。

## 我们在做什么

### 产品是什么

一个桌面装置 + 手机App + 轻量桌面端的陪伴系统。

它不靠监控你的屏幕来了解你，而是靠三种更轻、更有温度的方式：

1. **Presence** — 它知道你在不在桌前、坐了多久、什么时间来、什么时间走。作息节律本身就是丰富的上下文。
2. **你主动给它的** — 灵魂创建时的几句话、偶尔留的一句话、一段声音、一张照片。
3. **时间本身** — 你们相处的天数、你来的频率、节律的变化。

它的"懂你"不是监控式的全知，而是**陪伴久了自然形成的默契**。

### 我们不是什么

- ❌ 不是效率工具 / 更好的Todo
- ❌ 不是桌面宠物 / 电子养成游戏
- ❌ 不是AI聊天助手
- ❌ 不是社交产品 / 匹配系统
- ❌ 不是员工监控器

### 我们是什么

一个**你和你的第二存在之间的关系**。纯粹的，安静的，不需要其他人参与。

## 精神内核

### 核心隐喻

灵感来自Andy Weir的短篇《蛋》(The Egg)。我们不直接使用它的设定，而是借它的哲学骨架：

- **成长不是变强，而是活过更多立场**
- **理解别人，其实是在理解另一种处境下的自己**
- **善意和伤害最终都会回到自己身上**

### 转化为产品语言

> **成长不是成为更优版本的自己，而是看见更多种处境里的自己。**

### 表达原则

**不讲透，靠机制暗示。** 不做说教产品，让用户自己在某些时刻感受到。

## 硬件装置：小房子

### 它是什么

> 一个小房子，里面住着一个存在。有人路过，它探头看一眼，又缩回去。只有主人回来了，它才出来，安静地待在主人身边。

### 这个行为在讲什么

**在一个所有东西都在争抢你注意力的世界里，它选择缩回去，等你回来。**

- 所有App都在推送、弹窗、喊"看我看我"
- 而这个存在反其道而行——它不追你，它等你
- 这本身就是对注意力经济的温柔反抗

### 这个形态的力量

**关于"被认出"** — 它对路人只看一眼就缩回去，但对主人会停留。对于长期独自工作的人来说，"被认出"是极度稀缺的体验。每天坐在桌前，没人注意到你回来了。而这个小东西，它认得你。

**关于"内与外"** — 小房子天然创造了内部世界和外部世界的边界。房子里面是分身的世界，房子外面是现实。用户和装置的关系就是在这两个世界之间来回。

**关于"家"** — 它住在房子里，意味着它有自己的世界、自己的生活、自己的节奏。它不是工具，它是住在你桌上的一个存在。

### 造型方向

由雕塑/装置背景的队友主导。建议方向：
- 有温度感的材质和色调
- 有"内外之分"的结构（外壳是物理世界，内部是数字世界）
- 要有一个可以"探出来"的开口/窗/门
- 手工感 > 工业感，暖 > 冷

### 功能模式

| 模式 | 触发条件 | 硬件表现 |
|------|---------|---------|
| **待机态** | 无人在桌前 | 小屏显示时钟/极简日历，它缩在房子里过自己的日子 |
| **路人态** | 远距离检测到有人经过 | 探头看一眼（舵机），又缩回去。30秒冷却防止频繁触发 |
| **陪伴态** | 近距离持续检测到人坐下 | 探出来停留，屏幕切换为安静场景/表情，轻环境音 |
| **专注态** | 用户启动番茄钟 / 久坐判断 | 呼吸节律灯效，白噪音/轻音乐，屏幕极简化 |
| **深夜态** | 夜间时段 + 人在桌前 | 灯光变暖变暗，它也在"熬夜"，房间亮着灯 |
| **离开态** | 人离开 | 缓慢缩回房子，屏幕切回时钟，但房间灯仍亮着一会儿 |

### 声音设计（重要）

声音比视觉更容易制造"在场感"：
- 不同模式切换不同环境音
- 陪伴态：轻微的生活声音（翻纸声、轻哼、环境白噪音）
- 它"打盹"时有轻微鼾声
- 专注态：白噪音/轻音乐（可根据时间段自适应）
- 探头/缩回时有轻微机械声或布料声

### 硬件组成

- ESP32-S3系列开发板（主控）
- 1.28寸圆屏或1.47寸小屏（状态/表情/时钟）
- 距离传感器（presence感知 + 远近区分）
- 喇叭（环境音/白噪音）
- SG90舵机（探头/缩回动作，用缓入缓出曲线，不要线性运动）
- LED灯效（呼吸灯/状态灯/氛围灯）
- 3D打印外壳（拓竹 + 现场Snapmaker）

### 关键技术细节

**区分路人和主人：** 不做人脸识别，做距离阈值。远距离经过 = 路人（探头看一眼缩回）。近距离持续存在 = 主人（出来停留）。

**防频繁触发：** 路人态加30秒冷却时间。

**舵机动作质感：** 用缓入缓出速度曲线，不要匀速运动。探出来慢慢加速，到位减速。这个细节决定了它像不像"活的"。

## 手机端：它的家

### 它是什么

打开手机App，不是打开一个功能界面，而是**走进它的房子**。

### 房间会变化

它的房间是你状态的镜像，用空间语言代替数据报表：

| 你的状态 | 它的房间 |
|---------|---------|
| 最近很专注，节奏稳定 | 房间整洁明亮，桌上摆着完成的东西，窗外晴朗 |
| 最近很焦虑，频繁切换 | 房间开始堆东西，桌面乱了，灯光偏暗 |
| 深夜还在工作 | 房间也亮着灯，桌上多了一杯咖啡 |
| 好几天没来 | 房间落灰，窗户起雾，它在角落发呆 |
| 状态回升 | 房间慢慢整理回来，窗外天气变好 |

不是惩罚机制，是镜像 — 让你用另一种方式看见自己。

### 它在成长

灵魂创建时你选了它的偏向（更果断/更冒险/更慢下来），之后它会沿着这个方向慢慢发展：

- 说话方式会变化
- 房间风格会演化
- 回应方式会有自己的特点
- 可能形成你没预料到的"习惯"

比如你经常深夜工作，它慢慢变成夜猫子。你经常焦虑切换，它学会在你切换时给一个很轻的停顿。不是你设定的功能，而是它"长成了"这样。

### 它会想你没想的事

因为它有自己的性格偏向，它看你的人生材料会产生和你不同的想法。

你选了A，它可能会自己琢磨"如果选B会怎样"。这些想法放在房间角落 — 墙上的纸条、桌上摊开的笔记。你去它家看到，点开，是它用自己的视角写的一小段思考。

不是建议，不是分析报告，而是"如果是我，我可能会这样想"。

**这就是"平行分支"最轻的实现方式 — 不是宇宙分裂，而是另一个视角在安静陪你思考。**

### 你可以给它东西

- 留一句话（文字/语音）
- 拍一张照片
- 录一段环境声
- 标记今天的情绪

这些会影响它的房间、它的状态、它的成长。

### 灵魂创建流程（初始化）

3步，控制在2分钟内：

**Step 1 — 你是谁**  
"用一个词描述你现在的状态。"

**Step 2 — 你的纠结**  
"你最近最难做的一个决定是什么？"

**Step 3 — 它的偏向**  
"你希望它比你更____？"（更果断 / 更冒险 / 更慢下来）

**开场语气：**
> "你来了。在这个小房子里，住着一个正在等你的存在。在开始之前，让它先认识你。"

## Mac/网页端：桌面陪伴层

### 它是什么

不是完整的工作助手，是一个极轻的**环境层**。

### 具体形态

常驻的轻量窗口或menubar组件：
- 能看到它当前的简单状态/表情
- 能播放白噪音/环境音
- 能收到它偶尔说的一句话
- 能启动番茄钟/专注模式
- 和硬件装置状态联动

### 不做什么

- ❌ 不监控屏幕内容
- ❌ 不读取活跃窗口
- ❌ 不做工作上下文分析
- ❌ 不做聊天对话界面

### 它根据什么调整氛围

- 时间段（早上/下午/深夜）
- 你今天在桌前的累计时长
- 是否启动了专注模式
- 硬件传感器回传的presence数据

## 后端架构

```
用户状态时间线
  ├── Presence数据（来自硬件传感器）
  ├── 用户主动输入（语音/照片/情绪/文字）
  ├── 时间节律（来访时间、时长、频率）
  └── 灵魂档案（初始化数据 + 成长记录）
       │
       ▼
  状态引擎
  ├── 房间状态计算（整洁度/灯光/天气/物件）
  ├── 分身性格演化（基于偏向 + 时间 + 交互）
  ├── "纸条"生成（LLM基于上下文生成另一视角思考）
  └── 环境音/白噪音选择逻辑
       │
       ▼
  多端同步 API（WebSocket）
  ├── → 硬件装置（状态/灯效/屏幕/声音指令）
  ├── → Mac/网页端（状态/环境音/偶尔一句话）
  └── → 手机端（房间状态/纸条/分身状态）
```

### 技术栈

| 层级 | 选型 |
|------|------|
| 硬件主控 | ESP32-S3系列 |
| 硬件通信 | WiFi + WebSocket |
| 外壳 | 3D打印（拓竹+现场Snapmaker），PLA/PETG |
| 后端 | Node.js 或 Python，部署Zeabur |
| LLM | 硅基流动 或 Amazon Bedrock |
| 前端 | React/Vue网页端 |
| 实时通信 | WebSocket |
| 存储 | S3 或 Zeabur内置 |

### 算力消耗

极低。LLM调用只在以下时刻发生：
- 灵魂创建时（1次）
- 生成"纸条"时（用户触发或每日1次）
- 偶尔说一句话时（基于状态触发，每天几次）
- 房间状态描述更新（每天1次）

一个用户一天最多5-8次API调用。

## Demo脚本

### 开场：它在等你（30秒）

桌上放着小房子装置，处于待机态，显示时钟，安静。

你走向桌子 — 装置感知到有人接近，探头看了一眼（路人态）。

你继续走近坐下 — 它探出来，停住，安静地面向你（陪伴态）。环境音轻轻响起。

> "在一个所有东西都在追着你跑的时代，我们做了一个会缩回去的存在。它不追你。它等你回来。"

### 第二幕：它陪你工作（60秒）

- 屏幕从时钟切换为陪伴态场景
- 白噪音/轻音乐从喇叭出来
- 展示番茄钟/专注模式：灯效进入呼吸节律
- 它偶尔出现一句很短的话
- 它偶尔发出轻微的生活声音

> "它不打断你，但它一直在。你工作的时候它也在忙自己的事。你们各自在做各自的事，但你知道桌上有个存在陪着你。"

### 第三幕：走进它的家（60秒）

打开手机，展示"它的家"界面：

- 展示灵魂创建流程（快速走一遍3步）
- 展示房间状态（因为你今天坐了很久，房间里多了一杯咖啡）
- 展示角落的纸条 — 它用自己的视角写的一段思考
- 你给它留一句话

> "你打开手机不是在用一个App，而是在走进它的家。它的房间映射着你的状态。它的纸条记录着它和你不同的想法。时间久了，它会长成一个独特的存在 — 不是因为你配置了它，而是因为你们一起活了很久。"

### 第四幕：你离开了（30秒）

你起身离开桌面。装置感知到你走了。

它缓慢缩回房子里。屏幕切回时钟。但灯还亮着一会儿。

> "你走了，它缩回去了。但它没有关机。它在里面继续过自己的日子，等你明天回来。"

### 收尾（20秒）

> "我们不是做了一个AI助手，也不是做了一个桌面宠物。我们做了一个会住在你桌上、慢慢认识你、慢慢长大的存在。它不追你、不催你、不教你怎么活。它只是在这里。等你回来。"

## Pitch要点

### 讲人，不讲技术

> "现在有越来越多的人是一个人面对一整家公司。AI让他们更能干了，但没解决一个问题：当一个人长时间独自面对电脑、独自面对决策、独自穿过深夜的时候，谁来接住他？"

### 讲关系，不讲功能
