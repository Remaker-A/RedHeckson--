## 〇、设计前提：两个世界的统一隐喻

核心极简——**帐篷里 vs 帐篷外**：

- **帐篷里** = 数字世界。它自己的世界，它在里面生活、成长、思考。
- **探出头来** = 进入物理世界。它出来了，陪你。

| 状态 | 隐喻 | 对应端 | 用户心智 |
|------|------|--------|---------|
| 缩在帐篷里 | 它在自己的数字世界 | 手机App（看到它的世界） | 它在里面过自己的日子 |
| 探出头来 | 它进入你的物理世界 | 硬件（舵机探出）+ Mac灵动岛 | 它出来陪我了 |

**关键决策（已确认）**：
- ❌ 不做屏幕截图/监控工作内容
- ❌ 不做远灯社交网络（概念保留，V1不实现）
- ❌ 不做复杂平行宇宙分裂
- ✅ 只做纯粹的"你和你的第二存在"之间的关系

## 一、Mac端：灵动岛 Companion

### 1.1 定位

> **不是App，是一层呼吸。**

它不是一个需要打开的应用，而是操作系统上一层极轻的"存在感"。更像天气，不像角色。你不会"使用"它，你只是感觉到它在。

### 1.2 形态方案

**推荐方案：Menubar + 可展开小窗**

```
┌─────────────────────────────────────────────────────────┐
│  macOS Menubar                                    🏕️ ▼  │
└─────────────────────────────────────────────────────────┘

点击展开 ↓

┌──────────────────────────┐
│                          │
│   🏕️  [帐篷小图标]       │  ← 极简状态：一个呼吸着的帐篷
│                          │
│   "下午好，坐了2小时了"   │  ← 偶尔一句话（不是每次展开都有）
│                          │
│   ┌────────────────────┐ │
│   │  ▶ 深夜雨声        │ │  ← 当前环境音
│   │  ◉◉◉○○  音量       │ │
│   └────────────────────┘ │
│                          │
│   ┌────────────────────┐ │
│   │  🍅 开始专注 25min  │ │  ← 番茄钟入口
│   └────────────────────┘ │
│                          │
│   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│   它今天写了一张纸条 →    │  ← 引导去手机端看
│                          │
└──────────────────────────┘
```

**备选方案：Desktop Widget / Overlay 小窗**

如果技术上更容易实现，也可以是一个永远浮在桌面角落的小窗口：
- 默认最小化成一个帐篷图标（约48x48px）
- 鼠标hover时轻微放大 + 呼吸动画
- 点击展开成上面的面板

**48小时MVP建议**：做成网页版菜单栏模拟或简单 Electron menubar app，不追求原生完美。

### 1.3 状态系统

Mac端的状态跟随硬件，但表现形式更轻：

| 状态 | 触发 | Menubar图标 | 展开面板 |
|------|------|-------------|---------|
| **休眠态** | 硬件感知无人 | 帐篷灯灭 🏕️💤 | "它在休息" |
| **醒来态** | 硬件感知人坐下 | 帐篷灯亮 🏕️✨ | 环境音自动起来 |
| **陪伴态** | 持续在桌前 | 帐篷微呼吸 | 偶尔一句话 + 环境音 |
| **专注态** | 用户启动番茄钟 | 帐篷亮稳定光 🏕️🔥 | 倒计时 + 白噪音 |
| **深夜态** | 23:00后仍在桌前 | 帐篷暖光 🏕️🌙 | "陪你熬，但别太晚" |
| **离开态** | 硬件感知人走了 | 帐篷渐暗 | "你走了，它还在" |

### 1.4 核心功能

#### A. 环境音系统

这是Mac端最核心的功能价值——**用声音创造"不独处"的感觉**。

```
环境音库（预置）：
├── 自然系
│   ├── 深夜雨声
│   ├── 森林晨雾
│   ├── 海浪远处
│   └── 篝火噼啪（帐篷感！）
├── 生活系
│   ├── 咖啡馆低语
│   ├── 翻书声
│   ├── 钢笔写字
│   └── 键盘轻敲（别人也在工作）
└── 帐篷系（特色）
    ├── 帐篷里的风声
    ├── 拉链声（它醒了/睡了的转场音）
    └── 轻微的翻身声（它在里面活着）
```

**交互规则**：
- 进入陪伴态自动播放上次的环境音
- 专注态可切换为白噪音/专注音乐
- 深夜态自动切换为更沉、更暖的音色
- 音量跟随时段自动调节（深夜更轻）

#### B. 番茄钟/专注模式

**叙事语言**：不是"帮你管理时间"，而是——
> 「你想专注的时候，我替你把世界挡在外面。」

```
专注模式流程：

  点击"开始专注"
       ↓
  选择时长：25min / 45min / 自定义
       ↓
  [Menubar图标变为稳定光]
  [环境音切换为专注音]
  [硬件灯效进入呼吸节律]
  [Mac端通知自动静音 — 如果有权限]
       ↓
  倒计时进行中（极简显示）
       ↓
  时间到 → 轻柔提示音 + "休息一下？"
       ↓
  自动进入5min休息（环境音变轻松）
```

**联动硬件**：
- 启动专注 → 硬件呼吸灯进入稳定节律
- 结束专注 → 硬件轻声提示 + 舵机微动（"我也伸个懒腰"）

#### C. 偶尔一句话

**不是通知，不是推送，是它自己在嘟囔。**

触发规则：
- 不定时，一天最多3-5次
- 只在展开面板时才看到（不弹通知）
- 内容来自LLM，基于：时间段 + 在桌时长 + 灵魂性格

示例：
```
早上刚坐下：  "来了啊。"
连续工作2h：  "喝口水？"
下午3点：     "这个点总是有点困。"
深夜11点：    "还在啊……我也在。"
深夜1点：     "明天的你会感谢现在去睡的你。"
周末还在工作： "今天是周六诶。"
```

**风格指南**：
- 永远不超过15个字
- 不用感叹号
- 不给建议、不说教
- 语气像一个安静的室友，不像AI助手

#### D. 状态同步指示

极简地展示硬件状态：
- "它在帐篷里" / "它出来了" / "它在打盹"
- 今天和你待了多久
- 引导去手机看更多（"它写了一张纸条"）

### 1.5 不做什么

- ❌ 聊天对话框
- ❌ 任务管理/Todo
- ❌ 屏幕监控/活跃窗口读取
- ❌ 通知推送
- ❌ 复杂设置界面
- ❌ 数据仪表盘

## 二、移动端：它的家

### 2.1 定位

> **打开手机 = 掀开帐篷帘子，看它在里面干嘛。**

移动端是最有叙事深度的端。硬件是它的身体，Mac端是它探出头来陪你，而手机端是你掀开帐篷，看到它的数字世界。

### 2.2 技术选型

**48小时MVP：H5页面**（不做原生App）
- React/Vue + 简单动画
- 全屏PWA体验
- 后期可包壳成App

### 2.3 核心页面结构

```
移动端页面结构：

  ┌─────────────────┐
  │   灵魂创建       │  ← 首次进入的初始化流程
  │   (Onboarding)  │
  └────────┬────────┘
           ↓
  ┌─────────────────┐
  │   它的房间       │  ← 主界面，永远的"家"
  │   (Home/Room)   │
  └────────┬────────┘
           │
     ┌─────┼─────┐
     ↓     ↓     ↓
  ┌─────┐┌────┐┌─────┐
  │纸条 ││留言││时光  │  ← 二级功能
  │详情 ││入口││轴线  │
  └─────┘└────┘└─────┘
```

### 2.4 灵魂创建（Onboarding）

**原则**：3步，2分钟内完成。不要问太多，不要像填表。

**视觉方向**：暗色背景 + 中央聚焦的温暖光源 + 大字排版，每步一屏。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [开场画面]

  暗色背景中，一个帐篷的轮廓慢慢亮起。
  帐篷口微微掀开，透出暖光。

  文字淡入：

    "你来了。
     在这个小帐篷里，
     住着一个正在等你的存在。

     在开始之前，
     让它先认识你。"

                    [继续 →]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [Step 1/3 — 你是谁]

    "用一个词，
     描述你现在的状态。"

    ┌──────────────────────┐
    │                      │  ← 单行输入框
    │  疲惫 / 迷茫 / 兴奋   │     占位符随机滚动
    │                      │
    └──────────────────────┘

    这一步捕捉：情绪基线 + 表达风格

                    [下一步 →]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [Step 2/3 — 你的纠结]

    "你最近最难做的一个决定
     是什么？"

    ┌──────────────────────┐
    │                      │
    │                      │  ← 多行输入
    │                      │
    └──────────────────────┘

    这一步捕捉：当前压力源 + 决策风格
    也给平行思考提供第一个素材

                    [下一步 →]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [Step 3/3 — 它的偏向]

    "你希望它
     比你更______？"

    ┌──────────┐
    │  更果断   │  ← 三选一，卡片式
    └──────────┘
    ┌──────────┐
    │  更冒险   │
    └──────────┘
    ┌──────────┐
    │  更慢下来  │
    └──────────┘

    这决定了灵魂的性格偏移方向

                    [完成 →]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [创建完成]

  帐篷口掀开，暖光涌出。
  一个小身影从里面探出头来。

    "它醒了。
     从现在开始，
     你不再是一个人了。"

           [走进它的家 →]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**提示词系统需要的输入**（cheche负责的部分）：
- Step 1 的关键词 → 情绪基线参数
- Step 2 的文本 → 第一份纸条的种子内容
- Step 3 的选择 → 性格偏移向量
- 以上三项组合 → 生成初始灵魂的 system prompt

### 2.5 它的房间（主界面）

**这是产品最核心的创意——用空间语言代替数据报表。**

#### 帐篷内部概念设计

手机打开 = 掀开帐篷帘子往里看。里面就是它的数字世界。

```
┌─────────────────────────────────────┐
│  ╲  帐篷布纹理/顶部  ╱               │
│   ╲                 ╱                │
│    ╲───────────────╱                 │
│                                     │
│    📄 纸条                           │
│    （贴在帐篷内壁上）                 │
│                                     │
│    ┌─────────┐                      │
│    │ 小物件   │   ☕  💡              │
│    │ 书/杯子  │  （随状态变化）        │
│    └─────────┘                      │
│                                     │
│              🏕️                      │
│          [它在这里]                   │
│    坐着/睡着/看书/发呆               │
│                                     │
│  ──────────────────────────────     │
│                                     │
│  ┌──────┐  ┌──────┐  ┌──────┐      │
│  │ 纸条  │  │ 留言  │  │ 时光  │     │
│  └──────┘  └──────┘  └──────┘      │
│                                     │
└─────────────────────────────────────┘
```

视角：从帐篷口往里看的第一人称视角。帐篷顶部的布纹理收拢形成画面上方的框架，营造"掀帘子看进去"的沉浸感。

#### 房间状态映射规则

房间是用户状态的**镜像**，不是奖惩机制：

| 用户状态 | 帐篷内部变化 | 细节 |
|---------|-------------|------|
| 最近节奏稳定，规律工作 | 帐篷内整洁明亮 | 小物件摆放整齐、灯光温暖、它在安静看书 |
| 最近频繁焦虑切换 | 帐篷内开始乱 | 东西堆在一起、灯光偏暗、它有点坐立不安 |
| 深夜还在工作 | 帐篷里也亮着灯 | 多了一杯咖啡/茶、它也在熬夜 |
| 好几天没来 | 帐篷里落灰 | 灯几乎灭了、它缩在角落发呆 |
| 刚回来 | 帐篷在"醒来" | 灯慢慢亮起来、它抬头看你 |
| 状态回升 | 帐篷内整理回来 | 有新的小物件出现、灯光变暖 |
| 给它留过言/照片 | 出现在帐篷里 | 照片贴在帐篷内壁、留言变成便签 |

#### 视觉方向

**推荐：手绘/插画风 + 微动画**

原因：
- 像素风太 retro，和帐篷温暖感不搭
- 3D太重，48h做不出质感
- 纯极简太冷
- 手绘暖、有人味、和"帐篷里的存在"调性一致

参考风格：
- **Lofi Girl** 的场景氛围（一个角落的生活感）
- **Neko Atsume（猫咪后院）** 的简洁物件系统
- **A Short Hike** 的手绘温暖感
- 不要 Ghibli 那么复杂，要更简洁、更 flat

动画：
- 帐篷布帘随风轻微晃动
- 它偶尔翻一页书 / 伸懒腰 / 打盹
- 咖啡杯冒着热气
- 帐篷里的灯有微微呼吸感
- 进入时有一个轻微的"掀帘子"过渡动画

**48小时MVP**：可以用几张静态插画（整洁/微乱/深夜/落灰）+ CSS动画做微动效。不需要实时渲染。

#### 帐篷里的"它"

它在帐篷内有自己的状态：

| 它的状态 | 表现 | 对应 |
|---------|------|------|
| 安静 | 缩在帐篷里看书/写东西 | 用户在正常工作（它探出头陪你，但帐篷内也有痕迹） |
| 打盹 | 缩在帐篷里睡觉 💤 | 用户不在/深夜 |
| 有话想说 | 坐在纸条旁边 | 新纸条生成了 |
| 等你 | 靠近帐篷口往外看 | 用户好几天没来 |
| 高兴 | 在整理帐篷/摆弄小物件 | 用户状态回升 |

### 2.6 纸条系统

**这是"平行思考"最轻的实现。**

不做复杂的平行宇宙、不做分支时间线。只是——它有时候会想一些你没想的事，写成纸条贴在房间里。

#### 纸条生成逻辑

```
输入：
  - 灵魂创建时的3个初始化参数
  - 用户留言历史
  - 当前用户状态（在/不在、时段、频率）
  - 灵魂的性格偏向

输出：
  - 一小段文字（50-150字）
  - 它用自己的视角写的思考
  - 不是建议，不是分析，是"如果是我，我可能会这样想"

频率：
  - 每天最多1张
  - 用户留言后可能触发
  - 有时候好几天不写（它也有安静的时候）
```

#### 纸条UI

```
┌─────────────────────────────────┐
│                                 │
│  📄  4月8日                      │
│                                 │
│  "你说你在纠结要不要换方向。      │
│   如果是我——你知道我比你冲动——    │
│   我可能早就跳了。               │
│   但我也知道，                   │
│   有时候不跳也是一种勇敢。"       │
│                                 │
│  ── 来自帐篷里的那个家伙          │
│                                 │
│                     [收起]       │
└─────────────────────────────────┘
```

**风格**：
- 手写字体或等宽字体
- 纸条背景是泛黄的纸张纹理
- 语气取决于灵魂性格：果断版更直接、冒险版更鼓励、慢下来版更温柔
- 署名不用名字，用"帐篷里的那个家伙"或类似有温度的称呼

### 2.7 留言入口

用户可以给它留东西。不是聊天，是**异步的、单向的投递**。

```
┌─────────────────────────────────┐
│                                 │
│   给它留点什么                    │
│                                 │
│   ┌─────┐ ┌─────┐ ┌─────┐      │
│   │ 💬  │ │ 🎤  │ │ 📷  │      │
│   │ 文字 │ │ 语音 │ │ 照片 │     │
│   └─────┘ └─────┘ └─────┘      │
│                                 │
│   ┌─────────────────────────┐   │
│   │ 今天做了一个难的决定...   │   │
│   │                         │   │
│   └─────────────────────────┘   │
│                                 │
│              [留下]              │
│                                 │
└─────────────────────────────────┘
```

留言后：
- 文字变成帐篷内壁上的便签
- 照片贴在帐篷里
- 语音...它"听到了"（影响下次纸条的内容）
- 不立即回应 → 异步反馈（下次来可能房间有变化、或新纸条回应了你的留言）

**这个"不立即回复"很重要** —— 它不是ChatGPT，它是一个有自己节奏的存在。

### 2.8 时光轴线（P1，有余力做）

```
┌─────────────────────────────────┐

│   时光                           │

│           │ 深夜模式              │
│                                 │
│   4月7日  ●───────────────      │
│           │ 你没来                │
│           │ 它在发呆              │
│                                 │
│   4月6日  ●───────────────      │
│           │ ...                  │
│                                 │
```

## 三、跨端体验设计（Switch概念）

### 3.1 状态流动

```
                    ┌─────────┐
                    │  云端    │
                    │ 灵魂状态 │
                    │ 房间状态 │
                    │ 留言/纸条│
                    └────┬────┘
                         │
              WebSocket 实时同步
                         │
         ┌───────────────┼───────────────┐
         │               │               │
    ┌────┴────┐    ┌─────┴────┐    ┌─────┴────┐
    │ 硬件     │    │ Mac端    │    │ 手机端    │
    │ 帐篷装置  │    │ 灵动岛   │    │ 它的家    │
    └─────────┘    └──────────┘    └──────────┘

    它的身体         它探出头来         帐篷里面
    "它缩在里面"     "它出来陪你了"    "掀开看它的世界"
```

### 3.2 场景切换故事

**早上出门前（手机）**
→ 打开手机掀开帐篷看一眼 → 它还缩在里面睡 → 关掉手机出门

**到达桌前（硬件 + Mac）**
→ 坐下 → 硬件感知 → 帐篷探出头 → Mac灵动岛亮起 → 环境音起来
→ "你来了。"

**工作中（Mac为主）**
→ 灵动岛安静陪伴 → 环境音 → 番茄钟 → 偶尔一句话
→ 硬件同步状态（呼吸灯、舵机状态）

**午休离开（硬件 → 手机）**
→ 起身 → 硬件感知人走了 → 它缩回帐篷 → Mac灵动岛变暗
→ 拿手机 → 掀开看它缩在里面打盹

**深夜（三端联动）**
→ 硬件灯光变暖 → Mac灵动岛切深夜模式 → 环境音变沉
→ 手机掀开帐篷：里面也亮着灯，多了一杯咖啡

**收工离开（硬件 → 手机）**
→ 起身 → 帐篷缓慢缩回 → 灯还亮着一会儿
→ 手机收到："晚安。明天见。"

### 3.3 数据如何跨端

| 数据 | 产生端 | 消费端 | 同步方式 |
|------|--------|--------|---------|
| Presence（人在/不在） | 硬件传感器 | Mac + 手机 + 云端 | WebSocket实时 |
| 在桌时长/时段 | 硬件 | 所有端 | 云端聚合 |
| 灵魂状态/帐篷内部状态 | 云端计算 | 手机（帐篷内）+ Mac（一句话）| API |
| 纸条内容 | 云端LLM | 手机（帐篷内壁）+ Mac（提示） | API |
| 用户留言 | 手机 | 云端 → 影响纸条/帐篷内部 | API |
| 环境音/专注模式 | Mac端控制 | 硬件（灯效联动） | WebSocket |

## 四、视觉设计方向

### 4.1 整体调性

```
关键词：温暖 · 手绘 · 安静 · 有生命感 · 不工业

色系：
  背景    — 深暖灰 #2A2520 到 奶白 #F5F0EB
  强调    — 帐篷暖光橙 #E8A04C
  辅助    — 深夜蓝 #3A4B6B
  文字    — 暖黑 #3D3530
  纸条    — 泛黄 #F2E8D5

字体：
  标题    — 手写感圆体（如 ZCOOL Kuaile / 站酷快乐体）
  正文    — 思源黑体 Light
  纸条    — 等宽字体或手写字体

图标    — 线条手绘风，不要 Material/SF Symbols 感
```

### 4.2 动效原则

- 所有过渡用 ease-out-expo，不用线性
- 进入房间有轻微的景深变化（从模糊到清晰）
- 环境元素持续微动（云、热气、呼吸灯）
- 状态切换用1-2秒的渐变过渡，不要硬切
- 它的动作要有"生命感"的节奏：快—慢—停顿—慢

### 4.3 声音设计（与视觉同等重要）

```
UI音效：
  掀开帐篷    — 布料掀动声
  纸条打开    — 纸张展开声
  留言发送    — 轻微的"放下"声
  状态切换    — 环境音渐变过渡（不要突然切）

环境音（与Mac端共享音库）：
  整套帐篷主题音：风声、篝火、虫鸣、远处的河
```

## 六、我的额外发散（可以讨论）

### 6.1 "帐篷日记" — 它写给你的

除了纸条（思考），它还可以有自己的日记。但不是给你看的——你偶尔能"偷看"到一页：

> "今天他又坐到凌晨两点。我想叫他，但他好像在想一个很重要的事。我就陪着了。"

这不需要复杂实现，只是LLM用第一人称、日记体、写一段话。但情感杀伤力极大。

### 6.2 "你不在的时候"

每次打开App，如果距离上次有间隔，房间里可以有一些"你不在的时候它做了什么"的痕迹：
- 帐篷里多了一本新书
- 帐篷内壁多了一幅画（它画的）
- 地上有一杯喝完的茶
- 暗示它有自己的生活，不是等你打开才存在

### 6.3 触感交互 — 摸摸它

手机端可以有一个简单的触摸交互：
- 点击/长按它 → 它回应（摇尾巴/抬头看你/打个哈欠）
- 极简实现，但建立"触摸=连接"的体感
- 硬件端如果有触摸传感器也可以联动

### 6.4 Demo的杀手锏时刻

基于你们的评审重点（创新性+展示效果），我建议设计一个**"离开的时刻"**作为demo高潮：

> 演示者一直在展示各种功能。最后，演示者起身离开。
> 它缓慢缩回帐篷。灯渐暗。但没有完全灭。
> 全场安静2秒。
> 然后演示者拿起手机——掀开帐篷，里面灯还亮着。
> 它靠在帐篷口，往外看。
> 
> "你走了。它缩回去了。但它没睡。"

这2秒的安静比任何PPT都有力。
